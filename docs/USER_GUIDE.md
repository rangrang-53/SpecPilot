## 1. 시작하기

### 1.1 요구사항 확인

시작하기 전에 다음을 확인하세요:

- Python 3.11 이상 설치
- Google Gemini API 키 발급
- 안정적인 인터넷 연결

### 1.2 설치

```bash
# 저장소 클론
git clone https://github.com/your-username/SpecPilot.git
cd SpecPilot

# 가상환경 생성 및 활성화
python -m venv venv

# Windows
venv\Scripts\activate

# macOS/Linux
source venv/bin/activate

# 의존성 설치
pip install -r requirements.txt
```

### 1.3 환경 설정

```bash
# .env.example 복사
cp .env.example .env

# .env 파일 편집
# GOOGLE_API_KEY=your-api-key-here 설정
```

### 1.4 실행

```bash
streamlit run frontend/app.py
```

브라우저에서 `http://localhost:8501` 자동으로 열립니다.

## 2. 기본 사용법

### 2.1 세션 시작

#### Step 1: 프로젝트 아이디어 입력

초기 화면에서 프로젝트 아이디어를 입력합니다.

**좋은 입력 예시:**
```
온라인 의류 쇼핑몰을 만들고 싶습니다.
주요 기능:
- 상품 검색 및 필터링
- 장바구니
- 신용카드 결제
- 주문 내역 관리

예상 사용자: 월 1만명
```

**피해야 할 입력:**
```
쇼핑몰 만들기  (너무 간단함 - 최소 10자 이상)
```

#### Step 2: 시작하기 버튼 클릭

- 버튼: `🚀 시작하기`
- 단축키: `Ctrl + Enter`

### 2.2 AI 인터뷰 진행

AI가 프로젝트에 대한 추가 질문을 합니다.

**질문 유형:**

1. **인증 관련**
   - 예: "사용자 인증 방식은 어떻게 하시겠습니까?"
   - 추천 답변: "이메일/비밀번호 + 소셜 로그인(구글, 카카오)"

2. **배포 환경**
   - 예: "어떤 환경에 배포하실 계획인가요?"
   - 추천 답변: "AWS 클라우드"

3. **예상 규모**
   - 예: "예상 사용자 수는 얼마나 되나요?"
   - 추천 답변: "초기 1000명, 1년 후 1만명"

4. **결제 시스템** (이커머스인 경우)
   - 예: "어떤 결제 수단을 지원하시겠습니까?"
   - 추천 답변: "신용카드, 간편결제(카카오페이, 네이버페이)"

### 2.3 답변 입력 팁

#### 짧은 답변도 가능
```
✅ "AWS에 배포할 예정입니다"
✅ "카카오 로그인만 사용"
```

#### 상세한 답변 더 좋음
```
✅ "AWS에 배포할 예정이고, Docker 컨테이너로 패키징하겠습니다.
    CI/CD는 GitHub Actions를 사용하려고 합니다."
```

#### 모를 때는 건너뛰기
```
✅ "아직 정하지 못했습니다" 또는 "기본 옵션으로 해주세요"
```

### 2.4 SRS 문서 확인

충분한 정보가 수집되면 SRS 문서가 자동으로 생성됩니다.

#### 생성되는 항목:

1. **기능 요구사항 (Functional Requirements)**
   - ID: FR-001, FR-002, ...
   - 우선순위: High, Medium, Low
   - 기술 제안 포함

2. **비기능 요구사항 (Non-Functional Requirements)**
   - 성능, 보안, 확장성 등

3. **기술 스택 (Tech Stack)**
   - Backend, Frontend, Database, Infrastructure
   - 선택 이유(Rationale) 포함

4. **테스트 시나리오 (Test Scenarios)**
   - Gherkin 형식 (Given-When-Then)

### 2.5 문서 다운로드

#### 다운로드 형식:

1. **Markdown (.md)**
   - 가독성 좋음
   - GitHub, Notion 등에서 바로 사용 가능

2. **JSON (.json)**
   - 프로그래밍 방식으로 파싱 가능
   - API 통합에 유용

## 3. 고급 기능

### 3.1 세션 관리

#### 세션 ID 확인
- 사이드바에서 현재 세션 ID 확인 가능
- 예: `a1b2c3d4...`

#### 새 세션 시작
- 사이드바 하단 `🔄 새 세션 시작` 버튼 클릭
- 또는 완료 화면에서 `+ 새 프로젝트` 클릭

### 3.2 대화 내역 보기

완료 화면에서 `← 대화 보기` 버튼을 클릭하면:
- 모든 질문-답변 히스토리 확인
- SRS 생성 과정 추적

### 3.3 Iteration Counter

사이드바에서 현재 반복 횟수 확인:
- 최대 10회까지 질문
- 진행도를 시각적으로 표시

## 4. 문제 해결

### 4.1 일반적인 오류

#### "API Key가 유효하지 않습니다"

**원인:** Google Gemini API 키 설정 오류

**해결방법:**
1. `.env` 파일 확인
2. `GOOGLE_API_KEY=your-key-here` 형식 확인
3. API 키에 공백이나 따옴표 없는지 확인
4. [Google AI Studio](https://makersuite.google.com/app/apikey)에서 새 키 발급

#### "입력이 너무 짧습니다"

**원인:** 최소 10자 이상 입력 필요

**해결방법:**
```
❌ "쇼핑몰"  (5자)
✅ "쇼핑몰을 만들고 싶습니다"  (13자)
```

#### "세션을 찾을 수 없습니다"

**원인:** 세션이 만료되었거나 서버 재시작

**해결방법:**
1. 페이지 새로고침
2. 새 세션 시작

### 4.2 성능 최적화

#### LLM 응답이 느릴 때

1. **모델 변경**
   ```bash
   # .env 파일
   MODEL_NAME=gemini-1.5-flash  # pro 대신 flash 사용
   ```

2. **온도 조절**
   ```bash
   TEMPERATURE=0.5  # 기본값 0.7에서 낮춤
   ```

#### 메모리 사용량 높을 때

- 주기적으로 브라우저 새로고침
- 사용하지 않는 세션 삭제 (향후 버전에서 지원 예정)

### 4.3 디버깅

#### 로그 확인

```bash
# 터미널에서 로그 확인
# INFO 레벨 로그가 출력됨
```

#### 디버그 모드 활성화

```bash
# .env 파일
DEBUG=True
LOG_LEVEL=DEBUG
```

## 5. Best Practices

### 5.1 효과적인 프로젝트 설명 작성

#### Do's ✅

```
온라인 도서 대여 플랫폼을 만들고 싶습니다.

주요 기능:
- 도서 검색 및 예약
- 대여 기간 관리
- 연체료 자동 계산
- 모바일 앱 지원

예상 규모: 초기 500명의 회원, 3000권의 도서
```

#### Don'ts ❌

```
앱 만들기
```

### 5.2 질문 응답 전략

#### 1단계: 핵심만 먼저 답변
```
Q: 인증 방식은 어떻게 하시겠습니까?
A: 이메일/비밀번호 방식
```

#### 2단계: 추가 요청 시 상세 설명
```
Q: 소셜 로그인도 필요하신가요?
A: 네, 카카오와 구글 로그인 추가하고 싶습니다
```

### 5.3 프로젝트 유형별 팁

#### 이커머스
필수 답변 항목:
- 결제 수단 (카드, 간편결제 등)
- 재고 관리 방식
- 배송 추적 여부

#### 실시간 서비스 (채팅, 게임 등)
필수 답변 항목:
- 동시 접속자 수
- 실시간 통신 방식 (WebSocket, Polling)
- 지연시간 요구사항

#### 관리 시스템 (ERP, CRM 등)
필수 답변 항목:
- 사용자 권한 레벨
- 보고서/통계 기능
- 데이터 백업 정책

### 5.4 SRS 활용 방법

#### 개발 팀과 공유
1. Markdown 다운로드
2. GitHub Repository Wiki에 업로드
3. 팀원들과 리뷰

#### Jira/Trello 연동 (수동)
1. JSON 다운로드
2. 각 Functional Requirement를 Epic/Story로 변환
3. Priority에 따라 스프린트 계획

#### 기술 스택 검증
1. 생성된 Tech Stack 확인
2. 팀의 기술 스택과 비교
3. 필요시 수정 요청 (새 세션으로 재시도)

## 6. FAQ

### Q1: 한 번 생성한 SRS를 수정할 수 있나요?

**A:** 현재 버전(v1.0)에서는 직접 수정 기능이 없습니다.
새 세션을 시작하여 더 상세한 정보를 제공하면 더 정확한 SRS를 생성할 수 있습니다.

**향후 버전:** SRS 편집 기능 추가 예정

### Q2: 여러 프로젝트를 동시에 작업할 수 있나요?

**A:** 브라우저 탭을 여러 개 열면 독립적인 세션으로 동시 작업 가능합니다.

### Q3: 생성된 SRS를 팀원과 공유하려면?

**A:**
1. Markdown 파일 다운로드
2. GitHub Gist, Notion, Google Docs 등에 업로드
3. 링크 공유

### Q4: 영어로 된 프로젝트도 가능한가요?

**A:** v1.0은 한국어 최적화되어 있습니다. 영어 입력도 가능하지만 품질이 낮을 수 있습니다.

**향후 버전:** 다국어 지원 예정

### Q5: 생성된 기술 스택이 마음에 들지 않으면?

**A:** 초기 입력 시 선호하는 기술 스택을 명시하세요.

**예시:**
```
"온라인 쇼핑몰을 만들고 싶습니다.
백엔드는 Node.js를 사용하고, 프론트엔드는 React를 사용하려고 합니다."
```

### Q6: 무료로 사용할 수 있나요?

**A:**
- SpecPilot: 오픈소스, 무료
- Google Gemini API: 무료 티어 제공 (일일 제한 있음)
- 대량 사용 시: Gemini API 유료 플랜 필요
